# コントリビューションガイド

「ぶーぶーランド」への貢献ありがとうございます。
このプロジェクトは **GitHub Pages で動く静的サイト**です（中心は `index.html`。`manifest.webmanifest` / `service-worker.js` のほか、必要に応じて音源・画像・分割した JS/CSS などを同梱してよい）。

## 前提

- 原則ビルド不要・依存パッケージなしで動きますが、必要ならローカルでビルドして生成物をコミットしても構いません。
- 編集の中心は `index.html`（アプリ本体・スタイル・データ・SVG・音声を内包）。規模が大きくなったら別ファイルへ分割してもよい。

## ローカルでの動作確認

Service Worker は `https` か `localhost` でのみ動くため、ファイルを直接開くのではなく簡易サーバーで配信します。

```bash
python3 -m http.server 8000
# → http://localhost:8000 を開く
```

変更後は以下を確認してください。

- スタート →（名前入力）→ ホーム → 3モード → ガレージ が一通り動く
- GitHub Pages（サブパス配信）で動く（相対パスが崩れていない）
- ブラウザの DevTools コンソールにエラーが出ていない
- オフライン動作は任意（対応する場合はネットを切っても動くか確認）

Service Worker の構文は次でチェックできます。

```bash
node --check service-worker.js
```

## コーディング規約

- **素の HTML/CSS/JS を基本**とする。フレームワークやビルドは明確な必要がなければ入れない。外部CDN/APIへの実行時依存は避け、同一オリジン同梱を優先する。
- **GitHub Pages（静的ホスティング）で動く**こと。サーバーサイド処理を持たない。オフライン動作は努力目標（必須ではない）。
- **相対パス**（`./`）を使う。サブパス配信でも動くこと。
- **複数ファイル構成可**。単一HTMLに縛られず、音源・画像・分割 JS/CSS・ライブラリのローカルコピーを同梱してよい。
- **コンテンツは2歳児向け**: やさしい・短い・前向き・ひらがな中心。怖い／事故描写は不可。
- インデントや命名は周囲の既存コードに合わせる。

## コミット規約

[Conventional Commits](https://www.conventionalcommits.org/ja/) に従います。

```
<type>: <概要>

<本文（任意）>
```

- 主な type: `feat`（機能追加） / `fix`（バグ修正） / `chore`（雑務） / `docs`（ドキュメント） / `refactor`
- 概要は日本語で簡潔に。
- AI の支援を受けた場合は末尾に共著者を記載します。

```
Co-Authored-By: Claude <noreply@anthropic.com>
```

## バージョニング

[セマンティックバージョニング](https://semver.org/lang/ja/)（`MAJOR.MINOR.PATCH`）を採用します。現行は `v0.1.1`（`0.x` 系の開発初期）。

- **PATCH**（`0.0.x`）: バグ修正・軽微な調整
- **MINOR**（`0.x.0`）: 後方互換のある機能追加
- **MAJOR**（`x.0.0`）: 大きな仕様変更（`1.0.0` で初の安定版）

リリース時は次の **2 箇所を一緒に更新** します。

| 場所 | 役割 |
|---|---|
| `index.html` の `APP_VERSION` | スタート画面に表示するバージョン |
| `service-worker.js` の `CACHE` | キャッシュ世代。更新すると古いキャッシュが破棄され、新版が確実に配信される |

### リリース手順

1. `APP_VERSION` と `CACHE` を新しいバージョンに更新（例: `v0.1.2` / `buubuu-v0_1_2`）
2. コミットする（例: `chore: v0.1.2`）
3. `main` に push すると GitHub Pages へ自動反映される
4. （任意）git タグを付ける

```bash
git tag v0.1.2
git push --tags
```

> SW のキャッシュ世代を上げ忘れると、ホーム画面から開いた PWA に旧版が残ることがあります。バージョンと一緒に必ず更新してください。

## ブランチ / プルリクエスト

1. `main` から作業ブランチを切る（例: `feat/add-vehicle`）
2. 変更し、上記の動作確認を行う
3. プルリクエストを作成し、変更内容と確認結果を記載する
