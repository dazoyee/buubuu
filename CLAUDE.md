# CLAUDE.md

このリポジトリで作業する AI エージェント（Claude Code）向けの指針です。
人間の貢献者向けの詳細な手順・規約は [CONTRIBUTING.md](./CONTRIBUTING.md) を、概要は [README.md](./README.md) を参照してください。

## プロジェクト概要

「ぶーぶーランド」は2歳児向けの「はたらくのりもの」知育PWA。
中心の `index.html` にアプリ本体・コンテンツ・SVG・音声を内包し、`manifest.webmanifest` / `service-worker.js` を加えた静的サイト。必要に応じて音源・画像などの追加ファイルを同梱してよい。GitHub Pages（`dazoyee/buubuu`）で公開している。

## 制約（GitHub Pages 前提）

**GitHub Pages（静的ホスティング）で配信・動作すること**が唯一の必須要件。その範囲で以下を守る。

### 必須（守る）

- **静的ホスティングのみ**: サーバーサイド処理・自前バックエンド・DB を持たない。GitHub Pages に置いてそのまま動く形を保つ。
- **相対パス**: アセット参照・`start_url`・SW登録はすべて相対パス（`./`）。サブパス配信（`/buubuu/`）で動くこと。
- **コンテンツのトーン**: 2歳児向け＝やさしい・短い・前向き・ひらがな中心。怖い／事故の描写を入れない。

### 許容（GitHub Pages の範囲で自由）

- **複数ファイル構成可**: 「単一HTML／3ファイル完結」に縛られない。音源・画像・フォント・分割した `.js`/`.css`・ライブラリのローカルコピーをリポジトリに同梱してよい（同一オリジン配信）。
- **オフラインは任意**: 完全オフライン動作は必須要件ではない（努力目標）。Service Worker のキャッシュは「あってよいが必須ではない」。
- **ビルド工程可**: 必要ならローカルでビルドし生成物をコミットしてよい。

### 推奨（必須ではないが望ましい）

- **実行時の外部ホスト依存は最小化**: プライバシー・信頼性のため、外部CDN/API/フォント等への実行時依存はできるだけ避け、同一オリジン同梱を優先する。
- **依存は最小限**: フレームワーク（React 等）やビルドは、明確な必要がなければ入れない。素の HTML/CSS/JS を基本とする。

## 主要データの場所（`index.html` 内）

- `VEHICLES`: のりもの54種（名前・オノマトペ・効果音タイプ・やさしい一言 ほか）
- `VEHICLE_ART` / `vehicleBody()` / `buildVehicleSVG()`: のりもののアニメ付きSVGイラスト生成
- `STORIES`: 名前差し込み式のお話38話（`{name}` プレースホルダ、`short`／`long`）
- `SOUND_PARAMS` / `playSound()`: Web Audio による効果音合成
- `APP_VERSION`: スタート画面に表示するバージョン

## AI 特有の作業ガイド

- **描画の確認**: UI/SVG を変更したら、ヘッドレス Chrome でスクリーンショットを撮り、ランタイムエラーが無いことと見た目を確認する。
- **構文チェック**: `index.html` の `<script>` を抽出して `node --check`、`service-worker.js` も `node --check`。
- **データ整合**: のりもの・お話・効果音の参照（`id`／`shape`／`sound`）が一致しているか検証する。
- **リリース時のバージョン更新**: `index.html` の `APP_VERSION` と `service-worker.js` の `CACHE` を必ず一緒に更新する（詳細は CONTRIBUTING.md の「バージョニング」を参照）。
- **コーディング規約・コミット規約**: [CONTRIBUTING.md](./CONTRIBUTING.md) に従う。
